Abstract Script Engines

Abstract Script Engines, or ASE, is a cross-platform and language-independent scripting environment. ASE has an in-process communication mechanism which allows objects to communicate together across language boundaries. Supported languages and runtimes for this release include C++, Java, COM, .NET, Lua, Perl, SpiderMonkey, and Rhino.

Features:

  • Objects are usable across language boundaries. No IDL file is needed.
  • Allows to create dynamic proxies for statically typed interfaces. A Perl object can implement a Java interface, for example.
  • Exceptions can be thrown across language boundaries. A C++ exception can be caught by Java, for example.
  • Library DLLs are usable without any sort of registration.
  • Supports multithreading.
  • Script objects are accessible from C++ code via a single abstract datatype named ASE_Variant. C++ objects can be exposed to scripts with minimal effort, too. No explicit memory management is required.

Installing ASE: installer for Windows

  • Download asesetup from the ASE download page and install it.
  • If you want to use Java with ASE, you need to install JRE version 6 or later. You can download it from http://java.sun.com/javase/downloads/index.jsp.
  • If you want to use Perl with ASE, you need to install ActivePerl 5.8.x. You can download it from http://www.activestate.com/Products/ActivePerl/.
  • If you want to use .NET with ASE, you need to install .NET Framework 2.0 or newer. Use Windows Update to install .NET Framework.

Installing ASE: source tarballs

Source tarballs can be downloaded from the ASE download page. There are some requrements for language engines to work with ASE.

  • SpiderMonkey must be build with JS_THREADSAFE. Thus you need NSPR to build SpiderMonkey.
  • Lua must be build with C++, so that Lua uses C++ exceptions instead of longjmp().

Running scripts

On Windows, "aserun.exe" and "aserunw.exe" are the executables for ASE-compatible scripts. The former opens a console and the latter does not. Here is an example. 

    C:\Program Files\ASE\samples>aserun genuuid.lua
    a91868b7-cada-4f62-9ba0-2ff5108fd041

On Unix OSes, "aserun" is the executable. 

    $ aserun genuuid.lua
    df97470d-0bab-40b5-9a64-d9a1aff3f68a
